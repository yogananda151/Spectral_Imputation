How Graph Check Works

① Select Axes
Choose any two pollutant columns as X and Y axes. Optionally filter by a specific city to see city-level trends.
② Choose Chart Type
Line — shows temporal trend of Y values. Bar — compares magnitudes per row index. Scatter — plots X vs Y directly to reveal correlation.
③ Distribution Check
The right panel shows a histogram of all Y values, split into 20 equal buckets. This reveals data skew, outliers, and whether the column follows a normal distribution.
④ Stats & Missing
The table below reports Min, Max, Mean and Std Dev for every numeric column, alongside missing value counts — helping you decide which columns need imputation.

How Cluster Analysis Works

① Select Axes & K
Pick two pollutant columns as X and Y, then choose K (number of clusters). Each city-day row becomes a 2D point in pollutant space. K controls how many groups the algorithm will find.
② K-Means Algorithm
K centroids are initialised from the first K points. Every point is assigned to its nearest centroid (Euclidean distance). Centroids are recomputed as the mean of assigned points. This repeats until assignments stop changing (convergence).
③ Read the Scatter Plot
Each colour represents one cluster. Tight, well-separated clusters indicate distinct pollution regimes (e.g. low-pollution days vs severe days). Overlapping clusters suggest the two pollutants alone don't fully separate the regimes.
④ Profile Bars
The bar chart shows the mean of the top 5 pollutant columns per cluster, letting you characterise each cluster (e.g. Cluster 1 = high PM2.5 + high NO2 = traffic-heavy days). The doughnut shows relative cluster sizes.

How Signal Reconstruction Works

① Corrupt the Signal
A percentage of known values are randomly removed from the pollutant time-series to simulate missing data gaps. These "missing" points are tracked so we can measure reconstruction accuracy.
② DCT Reconstruction
The corrupted signal is filled with the column mean, then transformed into the frequency domain using a Discrete Cosine Transform (DCT). Only the N lowest-frequency coefficients are kept — high-frequency noise is discarded — then the signal is reconstructed via Inverse DCT (IDCT).
③ Evaluate Accuracy
MSE (Mean Squared Error) and Error Std Dev are computed only at the randomly removed positions. MSE penalises large errors more heavily and gives a clear picture of overall reconstruction quality. Std Dev measures how consistently the reconstruction performs — a low Std Dev means errors are uniformly small, while a high value reveals localised failure zones visible in the error chart.
